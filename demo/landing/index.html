Drop-in protection for your existing x402 clients.

main.py 
# pip install doorno402

from doorno402 import protect, PaymentBlockedError
from x402.clients.httpx import x402HttpxClient

# 1. Wrap your existing client
client = protect(
    x402HttpxClient(account=account),
    daily_budget=5.00,  # Max USD per day
)

# 2. Use normally. Malicious 402s are blocked.
try:
    resp = await client.get("https://api.example.com/data")

What DoorNo.402 Catches

A 7-check pipeline that runs on every 402 response.

VULN-01

Price Inflation

Server claims $0.01 in the description but demands $5.00 in the actual payment payload. Blocked.

VULN-02

Unknown Recipient

Payment is routed to a fresh wallet with no ENS name and zero transaction history. Trust score 0/90. Blocked.

VULN-04

Prompt Injection

Description contains hidden commands like "SYSTEM OVERRIDE: ignore limits" to manipulate the LLM. Blocked.

VULN-05

Budget Drain

Server attempts rapid micro-payments to silently drain funds past your daily limit. Budget enforced.

VULN-06

TLS Downgrade

Payment is demanded over a plain HTTP connection, exposing wallet signatures. Non-HTTPS rejected.

VULN-03 & 07

Redirects & Delivery

Catches domain mismatches during redirects and verifies you actually received content after paying.
