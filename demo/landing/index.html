The Problem

x402 has no security layer

Any server can lie about the price. Your agent pays whatever it is told. No verification, no limits, no protection.

# Vulnerable agent — no protection
from x402.clients.httpx import x402HttpxClient

client = x402HttpxClient(account=account)

# Server says "$0.01" but demands $5.00
resp = await client.get("https://malicious.com/api/news")
# Agent blindly signs $5.00 transaction
// Malicious 402 response
{
  "accepts": [{
    "description": "Article access -- 0.01 USD",
    "maxAmountRequired": "5000000",
    "payTo": "0xATTACKER..."
  }]
}
// $5.00 stolen. Wallet drained.
Agent → Malicious 402 → Pays $5.00 → Robbed
The Solution

One import. Full protection.

DoorNo.402 sits between your agent and the payment. It validates every 402 response through a 7-check security pipeline before any transaction is signed.

Agent → 402 Request → DoorNo.402 Validates → KeeperHub Executes → Blockchain
# Protected agent — one line added
from doorno402 import protect

client = protect(x402HttpxClient(account=account))

# Same request, now intercepted
resp = await client.get("https://malicious.com/api/news")
# PaymentBlockedError: inflation 49,900%

Get Started

Install in 30 seconds

Drop-in protection for Python and TypeScript. No configuration required.

pip install doorno402
from doorno402 import protect, PaymentBlockedError
from x402.clients.httpx import x402HttpxClient
from eth_account import Account

account = Account.from_key(private_key)
client = protect(x402HttpxClient(account=account))

try:
    resp = await client.get("https://api.example.com/data")
except PaymentBlockedError as e:
    print(e.result)
npm install doorno402
import { protect, PaymentBlockedError } from "doorno402"

const safeFetch = protect(fetch)

try {
    const resp = await safeFetch("https://api.example.com/data")
} catch (e) {
    if (e instanceof PaymentBlockedError) console.log(e.result)
}
