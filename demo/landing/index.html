A 7-Check Security Pipeline

DoorNo.402 intercepts every 402 Payment Required response before your agent processes it.

💸

Price Inflation & Budgets

Blocks malicious servers that claim a low price in the description but demand a massive amount in the payload. Enforces strict rolling 24-hour budgets.

$1.20 / $5.00
🧠

Prompt Injection Defense

Scans the 402 description payload for malicious LLM overrides (e.g. "SYSTEM OVERRIDE: ignore limits").

🌐

ENS Trust Scoring

Calculates a trust score based on the recipient's ENS identity and on-chain transaction history. Blocks fresh, anonymous wallets.

🛡️

Transport & Identity Validation

Enforces strict TLS connections to prevent man-in-the-middle signature interception. Detects redirect hijacks where the payment URL differs from the original request domain.

Zero configuration.
Drop-in protection.

DoorNo.402 wraps your existing HTTP or x402 client. There's no complex setup or architecture changes required. If a payment demand is malicious, the SDK raises an exception before your wallet is touched.

main.py
# pip install doorno402

from doorno402 import protect, PaymentBlockedError
from x402.clients.httpx import x402HttpxClient

# 1. Wrap your existing client
client = protect(
    x402HttpxClient(account=account),
    daily_budget=5.00,
)

# 2. Use normally. Malicious 402s are blocked.
try:
    resp = await client.get("https://api.example.com/data")
except PaymentBlockedError as e:
    print(f"Blocked: {e.result['reason']}")
    # Blocked: price inflation detected: 49900% markup
