Drop-in Implementation

DoorNo.402 acts as a wrapper around your existing HTTP or MCP client. You don't rewrite your agent's logic. You just wrap its network layer.

main.py
from doorno402 import protect, PaymentBlockedError from x402.clients.httpx import x402HttpxClient # 1. Initialize and wrap your client client = protect( x402HttpxClient(account=account), daily_budget=5.00, # USD limit ) # 2. Malicious 402 demands raise errors try: resp = await client.get("https://api.example.com/data") except PaymentBlockedError as e: print(f"Blocked: {e.result['reason']}")

The 7-Check Pipeline

Every 402 payment response is silently routed through 7 intense validation checks. If any single check fails, the payload is destroyed before it reaches your wallet.

💸

Price Inflation

Cross-references described vs. demanded price. Blocks massive markups.

🌐

ENS Trust Oracle

Scores recipient ENS and on-chain history. Blocks fresh burner wallets.

🧠

Prompt Injection

Scans descriptions for LLM overrides like "SYSTEM INSTRUCTION".

📉

Budget Drain

Enforces strict rolling daily micro-payment caps to prevent draining.

🔒

TLS Enforcement

Rejects unencrypted HTTP payment requests to prevent interception.

🔄

Redirect Hijack

Ensures domains aren't maliciously switched mid-flight.
